# iBOT FGVC Workflow

This repository is based on the official [iBOT](https://github.com/bytedance/ibot) codebase and is trimmed for the workflow used here:

- convert official DeiT checkpoints into an iBOT-compatible format
- continue iBOT pre-training from an ImageNet pre-trained backbone
- run linear probing on FGVC datasets such as CUB, Aircraft, and Cars

For the original paper and full upstream project details, see:

- [iBOT paper](https://arxiv.org/abs/2111.07832)
- [official iBOT repository](https://github.com/bytedance/ibot)

## Installation

See [INSTALL.md](INSTALL.md).

Important:

- run shell commands with `bash`, not `sh`
- the final `GPUS` argument in `run.sh` must match the number of visible GPUs
- for single-GPU runs, set `GPUS=1`

## Dataset Layout

FGVC datasets are expected in `ImageFolder` format:

```text
dataset/
  cub/
    train/
      class_0/
      class_1/
      ...
    test/
      class_0/
      class_1/
      ...
  air/
    train/
    test/
  cars/
    train/
    test/
```

The linear probing scripts support:

- `train` as the training split
- `test` as the evaluation split
- automatic class-count inference from the `train` folder

If the evaluation split is named `val`, set:

```bash
LINEAR_VAL_SPLIT=val
```

## Checkpoints

Local checkpoints used in this repository:

- `deit/deit_small_patch16_224-cd65a155.pth`: official DeiT-S/16 supervised checkpoint
- `deit/checkpoint_teacher.pth`: plain backbone checkpoint
- `deit/checkpoint_ibot_compatible.pth`: converted checkpoint with `teacher`, `student`, `state_dict`, and `model`

To regenerate the iBOT-compatible checkpoint:

```bash
python deit/transfer.py \
  --input deit/deit_small_patch16_224-cd65a155.pth \
  --output deit/checkpoint_ibot_compatible.pth
```

## Continue Pre-Training

This repository adds two arguments to `main_ibot.py`:

- `--pretrained_weights`
- `--pretrained_checkpoint_key`

They initialize the student backbone from an existing checkpoint before continued pre-training.

### CUB

```bash
CUDA_VISIBLE_DEVICES=0,1,2,3,4,5,6,7 \
INIT_PRETRAINED=/path/to/ibot/deit/checkpoint_ibot_compatible.pth \
INIT_CHECKPOINT_KEY=teacher \
OUTPUT_DIR=/path/to/ibot/output/cub_continue \
bash ./run.sh cub_pretrain cub_continue_vits16 vit_small teacher 8 \
  --epochs 100 \
  --batch_size_per_gpu 64 \
  --teacher_temp 0.07 \
  --warmup_teacher_temp_epochs 30 \
  --norm_last_layer false \
  --shared_head true \
  --out_dim 8192 \
  --local_crops_number 10 \
  --global_crops_scale 0.25 1 \
  --local_crops_scale 0.05 0.25 \
  --pred_ratio 0 0.3 \
  --pred_ratio_var 0 0.2
```

## Linear Probing

Use the checkpoint generated by continue pre-training:

```bash
$OUTPUT_DIR/checkpoint.pth
```

### CUB

```bash
CUDA_VISIBLE_DEVICES=0 \
PRETRAINED=/path/to/ibot/output/cub_continue/checkpoint.pth \
OUTPUT_DIR=/path/to/ibot/output/cub_continue/linear \
bash ./run.sh cub_linear cub_continue_vits16 vit_small teacher 1 \
  --epochs 100
```

### Aircraft

```bash
CUDA_VISIBLE_DEVICES=0 \
PRETRAINED=/path/to/ibot/output/air_continue/checkpoint.pth \
```

## Result Files

Linear probing logs are written to:

```bash
$OUTPUT_DIR/<dataset>_linear/log.txt
```

To extract the best top-1 accuracy:

```bash
python - <<'PY'
import json
from pathlib import Path

log_path = Path("/path/to/output/cub_continue/linear/cub_linear/log.txt")
best = 0.0
for line in log_path.read_text().splitlines():
    row = json.loads(line)
    for key, value in row.items():
        if key == "test_acc1" or (key.startswith("test_acc") and key.endswith("@1")):
            best = max(best, float(value))
print(best)
PY
```

## Notes

- `bash ./run.sh ...`, not `sh ./run.sh ...`
- single visible GPU means `GPUS=1`
- if `CUDA_VISIBLE_DEVICES=1`, the process still sees that device as local `cuda:0`
- if `timm` imports fail due to `apex` / old `torch` issues, this repository includes a fallback path in `models/vision_transformer.py` and `models/swin_transformer.py`
